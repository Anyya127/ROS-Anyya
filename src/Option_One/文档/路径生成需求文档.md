# waypoints.csv 生成需求

## 背景

一个四足机器狗仿真竞赛。赛道是 Gazebo 中的平面赛道，赛狗需要沿赛道中线自动行走。控制器使用 Pure Pursuit 算法，需要一条由 (x, y, yaw) 序列描述的中线路径。

赛狗已能在世界坐标系中获取自己的真值位姿，并能按给定速度前进和转向。唯一缺失的是正确的赛道中线路径点。

## 输出文件格式

**waypoints.csv**，每行三个逗号分隔的浮点数：

```
x,y,yaw
```

| 列 | 含义 | 单位 | 范围示例 |
|----|------|------|---------|
| x | Gazebo 世界坐标系 X 坐标 | 米 | -1 ~ 5 |
| y | Gazebo 世界坐标系 Y 坐标 | 米 | -1 ~ 17 |
| yaw | 机器人朝向角（从 +X 轴逆时针旋转） | 弧度 | -π ~ π |

**示例行**：
```
0.06,0.00,0.0        # 起点：狗 spawn 位置附近，头朝 +X（东）
0.06,0.50,1.57       # 转弯后朝 +Y（北）
1.38,1.00,1.57       # 沿直道向北
```

**要求**：
- 沿赛道中线，每隔 20~50cm 一个点，弯道密集、直线稀疏
- 相邻点间距 ≤ 0.5m
- 约 100-300 个点覆盖全长约 16m 的赛道
- yaw 值确保连续不跳变
- 不能踩黄线（赛道净宽 100cm，中线到边缘各 50cm）

## 世界坐标系与参考点

| 参照物 | 世界坐标 (x, y) |
|--------|----------------|
| **机器人 spawn 点** | **(0.0, 0.0)** ，头朝 +X |
| 赛道模型原点（race model） | (1.5, 5.87)，绕 Z 旋转 -1.57 rad (≈ -90°) |
| 地面高度 | z ≈ 0.0 |
| 悬挂球阵左下角 R1C1 | (-0.4, 1.34), z=0.3 |

| 悬挂球阵左上角 R4C1 | (-0.4, 3.86), z=0.3 |
| 悬挂球阵右上角 R4C4 | (3.2, 3.86), z=0.3 |
| football2（足球） | (2.1, 10.8), z=0.1 |
| coke（可乐瓶） | (-0.1, 11.1), z=0.1 |

## 坐标系说明

```
世界坐标系 (Gazebo):
        +Y (北)
        |
        |
        *---- +X (东)
       /
    +Z (上)

狗 spawn 位置: (x=0, y=0, z=0.6)
狗初始朝向: +X 方向 (yaw=0)
```

赛道模型 `race` 的位姿为 `(1.5, 5.87, 1.18)` 绕 Z 旋转 `-1.57 rad`。赛道模型的 STL 网格（来自 race2026_meshes 目录）在模型本地坐标系中定义，所有 mesh 缩放 0.01。模型本地 → 世界坐标的变换为：

```
世界X = mesh本地Y * 0.01 + 1.5
世界Y = -mesh本地X * 0.01 + 5.87
```

## 赛道布局参考

根据赛题，赛道分为 6 个赛段，沿 Y 轴方向延伸（从 Y≈0 到 Y≈16）。

已知悬挂球阵（赛段 2 "荒野寻珠"）在世界坐标系中占 y=1.34 到 y=3.86，x=-0.4 到 x=3.2。

大赛官方鸟瞰图显示了完整的赛道布局，包括各赛段的弯道、直道位置。请根据鸟瞰图识别赛道中线，结合以上参考点信息，将其转换为世界坐标系中的 (x, y, yaw) 序列。

## 验证标准

生成 CSV 后，在 Gazebo 中运行路径跟踪器 `./build/Option_One/follow_path`，狗应能：
1. 从 spawn 点自动站起
2. 沿赛道中线自动行走
3. 弯道自动转向
4. 不踩黄线
5. 到达赛道终点
