# 机器狗仿真竞赛 —— 方案一：Gazebo 真值定位 + 路径跟踪

## 一、方案总览

```
                    ┌─────────────────────────────┐
                    │      simulator_lcmt (LCM)    │
                    │   真值位姿: p[3], rpy[3]      │
                    └─────────────┬───────────────┘
                                  │
          ┌───────────────────────┼───────────────────────┐
          │                       │                       │
   ┌──────▼──────┐   ┌───────────▼────────┐   ┌──────────▼──────────┐
   │  路径跟踪器  │   │   视觉识别模块      │   │   全局状态机         │
   │ (PurePursuit)│   │ (摄像头 + OpenCV)  │   │   (赛段调度)         │
   │ 赛段 1,3 主力│   │ 赛段 2,4 主力      │   │   异常处理           │
   └──────┬──────┘   └───────────┬────────┘   └──────────┬──────────┘
          │                       │                       │
          └───────────────────────┼───────────────────────┘
                                  │
                    ┌─────────────▼─────────────┐
                    │  robot_control_cmd_lcmt    │
                    │  vel_des[3] (x,y,yaw)      │
                    │  mode, gait_id, life_count  │
                    └─────────────────────────────┘
```

**核心思路**：
- **定位**：订阅 `simulator_state` 获取 Gazebo 物理引擎的真值位姿 (x, y, yaw)，零漂移
- **路径**：手动遥控狗沿赛道中线走一圈，录制 waypoint 序列到 CSV 文件
- **控制**：Pure Pursuit 算法，实时根据位姿与目标路径偏差计算速度/角速度
- **指令**：通过 `robot_control_cmd_lcmt` 直接发送物理单位 (m/s, rad/s) 的速度指令

### 2.5 `robot_control_cmd_lcmt` 字段详解（⭐ 推荐接口）

```cpp
struct robot_control_cmd_lcmt {
    int8_t  mode;            // 控制模式 (11=locomotion)
    int8_t  gait_id;         // 步态 ID (3=trot medium, 27=trot slow)
    int8_t  contact;         // 四足触地状态
    int8_t  life_count;      // 心跳计数，值变化时命令才生效（安全机制）
    float   vel_des[3];      // [0]=x前向速度 m/s, [1]=y横向速度 m/s, [2]=yaw角速度 rad/s
    float   rpy_des[3];      // 期望 roll/pitch/yaw (rad)
    float   pos_des[3];      // 期望质心偏移 (m)
    float   acc_des[6];      // 跳跃加速度
    float   ctrl_point[3];   // 姿态控制中心点
    float   foot_pose[6];    // 足端偏移
    float   step_height[2];  // 抬腿高度 (0~0.08m)
    int32_t value;           // 附加参数
    int32_t duration;        // 持续时间 (ms), 0=持续执行
};
```

**为什么不用 `gamepad_lcmt`？**
- `gamepad_lcmt` 的摇杆值是 [-1,1] 的无量纲值，需要标定物理对应关系
- `robot_control_cmd_lcmt` 的 `vel_des[3]` 直接使用 **物理单位**（m/s, rad/s），更精准
- `robot_control_cmd_lcmt` 有 `life_count` 安全机制：命令超时 500ms 自动趴下保护
- `robot_control_cmd_lcmt` 优先级最高，不会被其他指令覆盖

## 三、各赛段详细分析

### 赛段 1：石径探路

| 项目 | 内容 |
|------|------|
| **任务** | 从起点出发 → 走石板路 → 转弯 → 通过弯道 |
| **难点** | 石板不平、弯道转向 |
| **核心技术** | 路径跟踪 |
| **实现方案** | 录制赛道中线 waypoint → Pure Pursuit 跟踪 → 结束时检测后腿是否越过弯道虚线 |
| **视觉需求** | 无 |
| **特殊动作** | 无 |

**实现细节**：
1. 机器狗从趴下状态站起（`control_mode=12` → `sleep 5s` → `control_mode=11`）
2. 开始 Pure Pursuit 路径跟踪，`vel_des[0]=0.25m/s`, gait=27 (慢 Trot)
3. 石板路部分：路径录制时自然包含地形变化，跟踪不受影响
4. 弯道部分：Pure Pursuit 自动调整角速度，内侧减速
5. 判断后腿越过弯道虚线 → 赛段结束

### 赛段 2：荒野寻珠

| 项目 | 内容 |
|------|------|
| **任务** | 在 4×4 悬挂球中，找到所有橙色球（每行每列各 1 个橙球，共 4 个），身体撞击使其明显晃动 |
| **难点** | 橙色球位置赛前随机调整；需要识别颜色；需要精准导航到球下方 |
| **核心技术** | 视觉颜色识别 + 导航 |
| **实现方案** | 导航到球阵区域 → 摄像头巡扫识别橙色球位置 → 逐个导航到球下 + 撞击 → 找到出口 |
| **视觉需求** | **必须**。识别悬挂球的颜色（橙色 vs 浅蓝色） |
| **特殊动作** | 身体撞击小球（正常行走高度即可触碰到距地 20cm 的球） |

**实现细节**：
1. 从赛段 1 出口，导航进入赛段 2 球阵区域
2. **球阵扫描**：在球阵前遍历，用摄像头拍下所有球，OpenCV 颜色分割识别橙色球，记录其行列位置
3. 已知 R4C4、R4C3、R3C4 固定为蓝色，只需要检查其余 13 个球
4. 逐个导航到橙色球下方 → 撞击（向前走一小段，让身体碰到球）
5. 撞击后后退，导航到下一个橙色球
6. 全部撞击完毕，导航到出口

> **待确认**：2025 赛道 world 文件中未包含 hanging_ball 模型。球阵的 world 定义需要确认。
> 可能需要从 race2024.world 中提取球阵模型，或大赛方会提供最终版 world。

### 赛段 3：曲道冲锋

| 项目 | 内容 |
|------|------|
| **任务** | 曲线→直线→曲线赛道，全程不踩线不越线 |
| **难点** | S 弯道，赛道仅宽 100cm，边线 15cm 黄线 |
| **核心技术** | 路径跟踪 |
| **实现方案** | 同赛段 1，录制中线 waypoint → Pure Pursuit |
| **视觉需求** | 无 |
| **特殊动作** | 无 |

**实现细节**：
- 这是 Gazebo 真值方案最强的场景。S 弯路径跟踪比开环控制精准 10 倍以上
- 可以适当提速（0.3~0.4 m/s），Pure Pursuit 自动在弯道减速

### 赛段 4：深隧寻珍

| 项目 | 内容 |
|------|------|
| **任务** | 3 条竖向通道中找：可乐瓶、橙色球、足球。避开：限高杆、不可跨越障碍。语音播报识别结果。 |
| **难点** | 目标物和障碍物赛前随机排列；需语音播报；可乐瓶需撞倒、足球需踢入门 |
| **核心技术** | 视觉识别 + 导航 + 语音输出 + 特殊动作 |
| **实现方案** | 遍历 3 条通道 → 每通道用摄像头识别 → 语音播报 → 执行对应交互 → 找到独木桥入口 |
| **视觉需求** | **必须**。识别可乐瓶、橙色球、足球、限高杆、障碍物 |
| **特殊动作** | 趴下通过限高杆、绕过障碍物、撞倒可乐瓶、踢球入门 |

**实现细节**：
1. 导航进入横向通道
2. 依次检查 3 条竖向通道：
   a. 进入通道 → 摄像头扫描
   b. 识别到物体 → 语音播报名称
   c. 根据物体类型执行交互：
      - 可乐瓶 → 撞倒（走上去碰倒）
      - 橙色球 → 撞击（球距地 60cm，可能需要跳起？或球悬挂高度狗身可触及）
      - 足球 → 推入球门
      - 限高杆 → 趴下通过（`control_mode=7` 趴下 → 匍匐前进 → `control_mode=12` 站起）
      - 不可跨越障碍 → 绕行（穿过虚线借道）
   d. 返回横向通道
3. 找到独木桥入口，前腿触碰到起始端 → 赛段结束

> **关于语音播报**：仿真环境中没有内置 TTS。可选方案：
> - A）使用系统 `espeak` / `festival` 命令输出语音
> - B）使用 Python `pyttsx3` 库
> - C）仅用 `std::cout` 打印文本（如果裁判接受）
> - D）确认大赛方对仿真中语音播报的具体要求

> **关于限高杆通过**：需要**趴下姿势通过**。
> - `control_mode=7`（PureDamper）是缓慢趴下，但趴下后无法行走
> - 可能需要自定义步态（降低机身高度 + 小步行走）
> - 或者使用 `pos_des[2]` 降低质心高度 + locomotion 步态
> **这是需要进一步研究的技术难点。**

> **关于撞击 60cm 高的橙色球**：球底部距地 60cm，狗站立时身体高度约 25-30cm。
> 直接身体撞击可能够不到。可能需要：
> - 跳跃模式（`mode=16 kJump3d` 或 `mode=22 kForceJump`）
> - 或者球实际上是悬挂的，底部距地 60cm 意味着球在头顶高度，狗可以用身体上部接触

### 赛段 5：孤梁稳渡

| 项目 | 内容 |
|------|------|
| **任务** | 全程在独木桥上行走，过虚线后跳下 |
| **难点** | 独木桥窄，易跌落；必须四足都越过虚线才能跳下 |
| **核心技术** | 精准路径跟踪 + 平衡控制 |
| **实现方案** | 独木桥中线 extremely narrow 路径跟踪 + 足端接触检测判断落桥 |
| **视觉需求** | 可选（辅助检测独木桥位置） |
| **特殊动作** | 跳下独木桥 |

**实现细节**：
1. 路径跟踪沿独木桥中线，速度极慢（0.1 m/s）
2. 使用 `state_estimator.contactEstimate[4]` 检测足端是否还在桥上
3. 一只脚落空 → 紧急停止/回退
4. 四足都越过虚线标志 → 跳下（`mode=16` Jump3d 或 `mode=22` ForceJump）
5. 跳下后站起

> **需要确认**：独木桥在 world 中的具体位置和尺寸。以及跳下动作的 jump mode 参数。

### 赛段 6：撷金建功

| 项目 | 内容 |
|------|------|
| **任务** | 将足球踢出出口 → 到达终点 → 四足在圈内 → 趴下 |
| **难点** | 足球位置可能不在正前方 |
| **核心技术** | 导航 + 踢球动作 + 精准定位 |
| **实现方案** | 导航到足球后 → 对准球门方向 → 踢球 → 导航到终点圈 → 趴下 |
| **视觉需求** | 可选（用于定位足球和球门） |
| **特殊动作** | 踢球、在圈内趴下 |

### 4.2 Pure Pursuit 算法

```
输入: 当前位姿 (x, y, θ), 路径点序列 waypoints[]
输出: 前向速度 v, 角速度 ω

1. 找到当前最近路径点 (closest point)
2. 从 closest point 向前搜索，找到距离机器人 ≥ L (前视距离) 的第一个点 = 前视点 (lookahead point)
3. 将前视点变换到机器人坐标系:
   dx = lookahead.x - x
   dy = lookahead.y - y
   x_local =  dx * cos(θ) + dy * sin(θ)
   y_local = -dx * sin(θ) + dy * cos(θ)
4. 计算曲率: κ = 2 * y_local / L²
5. 计算角速度: ω = v * κ
6. 弯道减速: if |ω| > threshold, v *= 0.6
7. 限幅: |ω| ≤ ω_max, 0 ≤ v ≤ v_max
```

**参数**（可调）：
- 前视距离 L: 0.4m
- 基础速度 v_base: 0.25 m/s
- 最大角速度 ω_max: 1.0 rad/s
- 终点容差: 0.15m
- 控制频率: 50Hz

### 4.3 全局状态机

```
States:
  INIT          → 初始化，加载所有赛段路径
  SEG1_START    → 赛段1: 站起 → 路径跟踪 → 检测出口
  SEG2_SEARCH   → 赛段2: 进入球阵 → 扫描识别橙球
  SEG2_HIT      → 赛段2: 逐个撞击橙球
  SEG2_EXIT     → 赛段2: 导航到出口
  SEG3_RUN      → 赛段3: 路径跟踪
  SEG4_EXPLORE  → 赛段4: 遍历通道 → 识别 → 交互
  SEG5_BEAM     → 赛段5: 独木桥行走
  SEG6_FINISH   → 赛段6: 踢球 → 到达终点 → 趴下
  EMERGENCY     → 异常处理（跌倒恢复、出界恢复）
  DONE          → 完成
```

### 5.4 悬挂球阵（赛段 2）

**Model: `hanging_ball`** — 16 个球 4×4 排列，球体 r=0.1m，通过 ball joint 悬挂，阻尼 0.5，距地约 0.3m。

球阵在**世界坐标系**中的位置（各行 y 坐标，各列 x 坐标）：

```
        C1 (x=-0.4)    C2 (x=0.8)    C3 (x=2.0)    C4 (x=3.2)
R1 (y=1.34)   蓝            橙             蓝             蓝
R2 (y=2.18)   蓝            蓝             橙             蓝
R3 (y=3.02)   蓝            蓝             蓝             橙*  ← 规则说应固定蓝
R4 (y=3.86)   橙*           蓝             蓝(固定)       蓝(固定)
```

> **颜色识别规则**：
> - 橙色 ambient=(0.92, 0.45, 0.17)，蓝色 ambient=(0.43, 0.74, 0.93)
> - 赛题规定：R3C4, R4C3, R4C4 三球固定为浅蓝色
> - **每行每列恰好 1 个橙色球**，其余为蓝色。赛前随机调整橙色球位置
> - **必须用摄像头识别橙色球，不可硬编码**

**⚠️ 注意**：world 文件中的球阵颜色是**一种随机示例**。比赛中橙色球位置会被重新调整。**我们的程序必须通过摄像头动态检测橙色球位置。**
